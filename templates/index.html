Github Data Visualization

Pavel Petrukhin

Description

The website analyzes the data from GitHub API for a given user. The idea is to obtain metrics that are important for each part of the software engineering process. The project not only evaluates size of changes, languages and technologies but also how the developer interacts with other developers. This is achieved by processing the data about issues and pull requests and looking at metrics such as average time it required the user to close the issue assigned to them or the average number of comments that was present in the issues and/or pull requests the user created.

The app serves as a great proof of concept for measuring Software Engineering. Even within the limits of college project, it is possible to gather a lot of data about developers, meaning that a dedicated team of software engineers could create a very effective measuring tool using GitHub API data.
